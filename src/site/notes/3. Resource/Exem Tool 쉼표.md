# 1. 개발배경
과거에 협업하는 분들의 휴가를 편리하게 파악하기 위해 구글 캘린더로 관리해왔는데요, 휴가 등록 이후 매번 수동으로 구글 캘린더에 관리하다 보니 번거로운 점이 많았습니다.

과거 방식 
![Pasted image 20260219221007.png](/img/user/Image/Pasted%20image%2020260219221007.png)
* 연차를 올리고 팀용 구글 캘린더에 등록
![Pasted image 20260219221144.png](/img/user/Image/Pasted%20image%2020260219221144.png)

번거롭기도 하지만 추가적으로 제가 꼼꼼하지 못한 편이라 이런 부분들을 잘 놓치는 경우가 많았습니다. ㅎ

그래서 그룹웨어 API에서 휴가자를 자동으로 추출하는 프로그램을 만들어 사용해왔고, 기존에는 환경변수로만 제어하는 방식이라 해당 프로그램 자체를 공유가 어려웠습니다.

초기 버전 프로젝트명 Exem Hook은 아래 설정 파일에서 제가 코프링으로 만든 서버 배포 시에 설정파일에 계정 정보, 챗봇 주소 등을 입력하여 배포하는 형식이였습니다. 
* 추후에 저희 AI 그룹은 해당 방식에서 하영지 과장님이 n8n으로 해당 로직 구현하여 조직 개편, 인원 추가 등 코드 방식보다는 유연하게 대응 할 수 있도록 사용했습니다. 
![Pasted image 20260219221601.png](/img/user/Image/Pasted%20image%2020260219221601.png)
만들고 나서 저만 사용하기 아깝다는 생각이들어 과거에 구글캘린더로 열심히 저희 일정을 관리해주시던 과거 팀장님에게 공유해드렸습니다. 
![Pasted image 20260219221837.png](/img/user/Image/Pasted%20image%2020260219221837.png)
팀장님의 피드백도 좋기도 했고 저와 장호형 차장님의 팀 뿐만아니라  "다른 팀에서도 이런 툴이 필요하지 않을까?" 하는 생각과 추가적으로, 주말 제외 공휴일 제외 등 피드백 주신 부분 수정하여서 팀장님께 다시 공유드리고 이러는 과정들이 번거로워 이번 설 연휴에 웹에서 직접 웹훅을 설정할 수 있는 서비스를 바이브코딩으로 만들어보는 것을 목표로 했습니다. 
# 2. 개발여정 & 활용한 AI 툴
앞서 개발 배경들을 모두 합치면(Exem hook + Exem Tool) 꽤 긴 여정이였겠지만 이번에 배포된 서비스로만 따졌을때 거의 실제 제가 넣은 시간은 2시간 정도이고 총 걸린 시간은 약 4시간 정도가 아니였나 합니다. 

실제 들어간 시간의 대부분은 요구사항을 작성하는대 사용했습니다.
제가 해결할 문제도 명확했고 어떻게 해결하고 하는 방향도 명확했습니다. 

기존의 Exem Hook은 어떤 그룹웨어의 어떤 API에서 휴가정보를 가져올 수 있는지 또 그룹웨어의 인증 방식등을 분석하고 테스트하는데 시간을 많이 소비했습니다.
Exem Hook은 그래도 6시간은 넘게 걸린 프로젝트 였던 것 같습니다. 

이번에 Exem Tool은 과거에 연동하는 방식의 코드와 요구사항의 초안을 만들고 요구사항문서도 계속 AI와 함께 초안을 다듬어갔습니다. 
![Pasted image 20260219223020.png](/img/user/Image/Pasted%20image%2020260219223020.png)
![Pasted image 20260219223121.png](/img/user/Image/Pasted%20image%2020260219223121.png)
그 이후에는 사실 클로드 코드에 해당 요구사항명세서와 기존의 그룹웨어와 연동하는 예시코드만 사용해서 너가 다 만들어라고 만든 뒤 클로드 코드를 기다렸습니다. 

클로드 코드가 만들어준 코드를 확인해보니 조직 & 인원 선택에 대한  요구사항을 제대로 이해하지 못한 부분이 있어서 아마 대부분 저의 요구사항이 명확하지 않아서 생기는 문제라고 생각해서 조금 더 디테일한 요구사항을 주기 위해 

- **조직도 드롭다운 탐색** — 부서 트리를 재귀로 펼치면서 최종 멤버까지 도달, 멤버는 name+position 표시, 선택 시 metadata.id 저장
라는 요구사항에 공식적인 명칭을 통해 조금더 AI에게 더 정확한 요구사항을 던져주니 제대로 구현을 했습니다. 
![Pasted image 20260219223827.png](/img/user/Image/Pasted%20image%2020260219223827.png)

케스케이딩 드롭다운과 Hierarchical Select 방식에 대해 알려주니 클로드 코드가 명확하게 제 요구사항을 이해하고 해당 요구 사항도 잘 달성했습니다. 
# 3. 개발 후 소감
이번 프로젝트 개발 후 소감은 AI 너무나도 모든 문제들을 잘해결한다라는 생각을 했습니다. 그리고 하루가 다르게 너무 빠르게 성능이 좋아져서 1달전에 해결안된 문제를 또 1달후의 AI에게 물어보면 해결되는 이런 일들이 일어나고 있는 것 같습니다. 

이번 개발은 과거의 개발 이력 부터 그룹웨어와 연동 방법 등 이전에 아직은 AI가 파악하기 어려운 개발을 이미 진행한 상태여서 완전한 바이브 코딩이라고 보기는 어렵지만 진짜 컨텍스트 아까운 수준의 수정사항이 아니면 저는 코드를 한줄도 작성하지 않았긴 합니다.
* 굉장히 간단한 서비스라 가능했던 것 같습니다. 

과거에는 개발자에게 가장 중요한 능력을 거시적으로 말하면 "문제 해결 능력"이라고 할 수 있었을 것 같습니다. 하지만 AI 시대에는 문제해결은 대부분의 영역에 있어서 AI가 저보다 다 좋은 결과를 가져다 주고 있다고 생각합니다.

이 시대에 가장 중요한 역량은 정해진 문제를 해결하는게 아니라 문제를 인식하고 정의하는 "문제 인식 능력"이 가장 중요한 것 같습니다. 정확하게 우리는 어떤 문제를 가지고 있는지 인식만한다면 대부분의 문제는 AI가 해결해주는 시대에 살고 있다.... AGI 시대가 오기전에 열심히 일해서 자산을 축적하자라는 생각을 했습니다. 


# 4. 앞으로 AI 활용해서 어떤 것들을 해보고 싶은지?
프로젝트 명이 Exem Tool인 이유는 사실 추가적으로 회사 내 생산성 관련된 프로그램들을 해당 프로젝트에서 추가적으로 진행해보고 싶어서 Exem Tool이라고 네이밍하여 앞으로도 개발을 통해 기존에는 리소스가 부족하여 못하던 사소한 문제들을 해결 해보고 싶습니다.

저는 제가 만든 프로그램을 누군가가 써주고 피드백을 주는것을 좋아하는편이라 이번 프로젝트가 생각보다 관심을 많이받아서 기분이 좋고 추가적으로 "exemble과 연동하여 회의 인원, 시간 등을 말하면 회의실을 예약해주는 서비스", "그룹웨어의 캘린더를 지속적으로 전송하는 서비스" 등 추가해보고싶습니다.

추가적으로 eXemble 관련해서 굉장히 요즘 바쁜 나날들을 보내고 있지만 언젠가는 여유가 된다면 정말 비즈니스적 가치가 있는 사이드 프로젝트도 개발해보고 싶습니다. 
eXemble도 지속적으로 피드백을 받고 개선해 나가고 있기 때문에 언젠가는 정말 eXemble이 회사의 많은 업무를 도와주는 솔루션이 될 수 있도록 개발해보겠습니다. 

마지막으로 eXemble 홍보를 한번만 해보면 eXemble의 추가적인 개발 사항들 모두 https://chat.exem.io/exem/channels/exem-exemble 해당 채널에서 공유하고 있으니 2주에 한번 정도 패치 진행될때만이라도 관심을 가져주시고.. eXemble이 어느정도 안정화된다면 위에서 말한것 처럼 엑셈에서 업무생산성을 높이는 Workflow를 만들어서 배포해보고 싶습니다.
